# Campaign Performance Database

## Purpose

This file defines how Accord Tech Solutions stores campaign results, winning sequences, reply reasons, booked meetings, and learning history.

Use this file after launch, during weekly review, and at campaign close.

---

## 01. Main Goal

The goal of the performance database is to turn every campaign into reusable learning.

The database should help the team answer:

- Which segments perform best?
- Which subject lines create replies?
- Which sequence steps create positive replies?
- Which CTAs work best?
- Which lead sources create bounce risk?
- Which inboxes perform well?
- Which offers should be reused or retired?

## 11. Learning Library

Store winning examples:

- Best subject lines
- Best first lines
- Best CTAs
- Best segment-specific hooks
- Best closing-loop emails
- Best stat hooks
- Best report/audit offers
- Best reply responses

Also store losing patterns:

- Weak subject lines
- High-bounce segments
- High-unsubscribe copy
- Bad-fit lead sources
- Confusing CTAs
- Overused stats

---

## 12. Final Rule

The database should be updated after every campaign.

Do not let campaign learning stay only inside Instantly. Export it, organize it, and use it to make the next campaign stronger.
